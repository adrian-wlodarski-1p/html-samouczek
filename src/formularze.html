Formularze

Formularze są jedną z najważniejszych funkcji HTML. Dziś się nauczymy, jak je tworzyć.

Znacznik <form>

Formularze tworzy się za pomocą <form>:

<form>
...
</form>

Można ustawić, co się stanie, gdy formularz zostanie uzupełniony, za pomocą atrybutów:

  • action - pozwala określić, gdzie formularz zostanie wysłany.
  • method - określa, w jaki sposób dane zostaną wysłane. Z get dane zostaną zawarte w URL, a z post zostaną one wysłane za pomocą HTTP.
  • autocomplete - czy dane będą automatycznie uzupełnione przez przeglądarkę? on znaczy tak, a off znaczy nie.
  • novalidate - atrybut logiczny, który sprawia, że formularz jest sprawdzany pod względem poprawności.
    <form novalidate>

Wysyłanie i resetowanie formularza

Są dwa sposoby, by stworzyć przycisk wysyłający formularz:

<input type="submit" value="..." />
<button type="submit">...</button>

Gdzie ... to tekst pokazywany na przycisku. Podobnie z przyciskiem resetującym formularz, tylko type="reset". submit jest domyślne w <button>.

Znacznik <input>

<input /> jest podstawowym znacznikiem samodzielnym w formularzach. Pokrywa on zdecydowaną większość rodzajów wejścia.

<input> ma różne atrybuty:

  • name identyfikuje element, pozwalając na odczytanie jego wartości po wysłaniu formularza. To NIE jest równoznaczne id.
  • value to domyślny tekst elementu.
  • readonly (logiczny) sprawia, że element jest tylko do odczytu.
  • disabled (logiczny) sprawia, że jakakolwiek interakcja z elementem jest niemożliwa. Nie może być on zaznaczony za pomocą Tab, nie jest on także wysyłany wraz z formularzem.
  • tabindex pozwala na określenie, który w kolejności zaznaczania jest przy pomocy Tab.
  • size teorytycznie wyraża "szerokość w znakach", w praktyce jednak to się mija z prawdą. Niezalecane, lepiej ustawić szerokość za pomocą CSS.
  • maxlength ustala maksymalną liczbę znaków, które można wpisać w pole.
  • required (logiczny) sprawia, że wybranie lub wpisanie wartości jest wymagane.
  • placeholder pozwala na wyświetlenie szarawego tekstu, który zniknie gdy użytkownik zacznie pisać.

<label>

Znacznik <label> pozwala opisać element formularza tekstem. Atrybut for="..." pozwala ustalić, elementowi z jakim id się przypisuje tekst.

<label for="imie">Imię: </label>
<input id="imie" name="imie" />

Atrybut type

Pole <input> może się zachowywać w różny sposób, w zależności od atrybutu type:

  • "text" (w. domyślna) pozwala na wpisanie tekstu.
  • "password" pozwala na wpisanie tekstu, który nie będzie widoczny na ekranie (zamiast tego będą widoczne kropki). Nie zaleca się przesyłania poufnych danych poprzez method="get".
  • W "email" można wpisać e-mail. Zawartość jest sprawdzana pod względem poprawności składniowej.
  • W "tel" można wpisać numer telefonu. Zawartość nie jest sprawdzana pod względem poprawności składniowej.
  • W "url" można wpisać URL strony internetowej. Zawartość jest sprawdzana. Zauważ, że wartość musi się zaczynać od https://.
  • W "date" można wpisać lub wybrać datę.
  • W "number" można wpisać liczbę. Zawartość jest sprawdzana. Można ograniczyć możliwe liczby poprzez atrybuty: min ustala minimalną wartość, max ustala maksymalną, a step, co iloma liczbami znajdują się prawidłowe wartości.
  • range jest analogiczne do number, ale ma formę suwaka:
  • color pozwala wybrać kolor:
  • hidden nie wyświetla się na stronie, ale jest wysyłane wraz z formularzem.
  • file pozwala wybrać pliki. Dodaj atrybut logiczny multiple, aby móc wybrać wiele plików.
  • image działa jak submit, ale ma wygląd obrazka wskazanego atrybutem src. Alternatywa to: 
    <button><img src="..." /></button>

Pola wyboru za pomocą <input>

  • type="checkbox" pozwala na zaznaczenie więcej niż jednej opcji. name określa indywidualną nazwę opcji, a value (opcjonalne) określa wartość, którą przyjmuje gdy działa: 
    <label for="check">Czy umiesz?</label>
<input type="checkbox" id="check" name="check" />
  • type="radio" pozwala na zaznaczenie tylko jednej opcji. name określa nazwę całego pola wyboru (i musi być takie same dla każdej opcji), a value określa nazwę opcji. Jako że odznaczenie opcji nie jest domyślnie możliwe, najlepiej umieścić przycisk resetujący wewnątrz formularza: 
    <label for="psy">Wolę psy</label>
<input type="radio" id="psy" name="preferencja" value="psy"/>
<label for="koty">Wolę koty</label>
<input type="radio" id="koty" name="preferencja" value="koty"/>

Inne elementy

  • <textarea> pozwala na wpisanie dłuższego tekstu. Atrybut cols określa szerokość, a rows - wysokość. Jest to znacznik kontenerowy, a wpisany w środku tekst jest początkową zawartością pola. Poniżej jest zastosowany atrybut placeholder.
  • <select> pozwala na stworzenie listy typu dropdown. Atrybut name jest nazwą listy. Wyborami są <option>, których value są nazwami opcji. Możesz dodać atrybut logiczny multiple, jeśli chcesz, by można było obliczyć wiele opcji. 
    <label>Wybierz opcję:
<select id="lista" name="lista">
  <option value="opcja1">Opcja 1</option>
  <option value="opcja2">Opcja 2</option>
  <option value="opcja3">Opcja 3</option>
</select></label>
  • <fieldset> pozwala na stworzenie ładnej ramki dla formularza. Znacznik podrzędny <legend> zawiera wyświetlaną nazwę formularza. 
    <form>
  <fieldset>
    <legend>...</legend>
    ...
  </fieldset>
</form>
